## 核心定位

**提示词工程1.0 = 信息维度建设**

- ✅ 做什么：盘点AI需要的所有信息、定义来源和更新机制
- ❌ 不做：判断标准、评估方法、识别技巧（这些是AI的推理能力）
- 🎯 目标：让AI拥有充分、准确、及时的信息

**信息维度 = AI需要的原材料**

- 原始数据/事实（用户原话、竞品功能、会议纪要）
- 公司特有知识（战略、术语、DJ观点）
- 专业领域知识（教育理论、学科知识）
- 历史经验（成败案例、问题记录）
- 资源现状（合作方、项目信息）

# 【M类】马斯洛需求层次库

**自动化方式**：AI基于马斯洛理论框架自动生成5个层次的细分维度 → 从会议纪要中提取DJ关于需求层次的讨论和观点 → 结合公司实践经验 → 形成公司特色的需求层次库

**为什么这是核心竞争力**：

## 维度迭代机制

### 补充触发

- **AI发现缺口**：AI执行任务时发现信息缺失 → 标注缺口 → 补充到对应维度
- **人发现遗漏**：人审核时发现AI未考虑的因素 → 主动添加新维度
- **项目复盘**：识别"如果当时有XX信息，质量会更高" → 补充新维度

### 质量管理

- **有效性检查**（每季度）：统计使用频率 → 删除/合并低频维度 → 重点维护高频维度
- **准确性检查**（每月）：检查信息是否过时 → 更新时效性信息 → 标注有效期
- **完整性检查**（项目复盘）：分析质量问题是否因维度缺失 → 补充新维度 → 调整颗粒度
